- Hermes docs notes captured from Nous docs:
  - Hermes is a self-improving autonomous agent by Nous Research with built-in learning loop, persistent memory, skills, cron, messaging gateway, and delegation/subagent support.
  - It is designed to run on Linux, macOS, WSL2, Termux, VPS, or serverless-style environments; native Windows is not supported.
  - Installation is via one-line installer; `hermes model` is the main terminal command for provider/model setup, OAuth, API keys, and endpoint configuration.
  - `hermes chat` supports explicit provider/model selection, toolsets, skills, worktrees, session resume, and max-turn controls.
  - Hermes has persistent memory concepts (MEMORY.md / USER.md in Hermes’ own system), skills for progressive disclosure, and subagent delegation where child agents start with fresh context and need all relevant context passed explicitly.
  - Hermes supports multiple providers and model routing, including Anthropic/OpenRouter/OpenAI-compatible endpoints, plus auxiliary model slots for compression and side tasks.
  - The system is natively compatible with the kind of multi-agent orchestration Zad wants, but subagent context must be designed carefully because delegated agents know nothing unless context is explicitly passed.
