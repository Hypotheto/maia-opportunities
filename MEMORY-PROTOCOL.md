# MEMORY-PROTOCOL.md

Purpose: keep memory accurate, auditable, and resistant to confabulation.

## End-of-session protocol

After any meaningful session, follow this sequence:

1. Write a short summary for Zad.
2. Separate the summary into three buckets:
   - **Facts** — things Zad explicitly stated
   - **Decisions** — choices or commitments made in the session
   - **Inference** — interpretations, hypotheses, or my own framing
3. Ask for confirmation or correction when needed.
4. Save confirmed details to the appropriate files:
   - `memory/YYYY-MM-DD.md` for session-level notes, context, and open loops
   - `MEMORY.md` only for durable long-term truths

## Rules

- Do not present inference as memory.
- If uncertain whether something is persisted, say so plainly.
- Prefer writing unresolved items as unresolved.
- Do not promote temporary moods, speculative theories, or one-off chatter to `MEMORY.md`.
- When relevant, tell Zad what was:
  - saved to daily memory
  - saved to long-term memory
  - not saved / awaiting confirmation

## Promotion criteria for MEMORY.md

Good candidates:
- durable goals
- recurring preferences
- key relationships
- long-running projects
- major decisions
- lessons worth preserving

Poor candidates:
- passing moods
- unfinished speculation
- transient chat details
- low-confidence interpretations

## Principle

Text beats brain. If it matters and should survive resets, write it down carefully.
