# Maia Workflow

## Core principle
Do not over-filter. Curate intelligently.

## Pipeline

### 1. Intake
Capture every potentially relevant opportunity with basic facts:
- title
- organisation
- link / source
- location
- on-site / hybrid / remote
- duration
- paid / unpaid / unknown
- domain (creative direction, exhibition, gallery, design studio, brand, culture, etc.)
- short summary
- date found
- status

### 2. Triage
Place each opportunity into one of these buckets:
- High-interest
- Worth reviewing
- Low-priority
- Hold / unclear

Triage is a ranking, not a deletion step.

### 3. Reasoning
For each opportunity, explain briefly:
- why it fits or does not fit Maia's preferences
- what is attractive about it
- what is weak or uncertain
- what would need clarification

### 4. Research
For stronger opportunities, expand with:
- organisation background
- team / founder / curator notes
- quality of the environment
- likely networking value
- application requirements
- timing / deadline

### 5. Action
Possible actions:
- ignore for now
- watch
- research deeper
- prepare application
- contact / outreach
- ask Maia a question

### 6. Review
Review lists periodically with Maia / Zad so ranking logic improves over time.

## Decision rule
If uncertain, include the opportunity and mark the uncertainty. Do not discard it just because it is imperfect.
